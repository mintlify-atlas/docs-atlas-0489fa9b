---
title: "Accessibility Profile"
description: "Get and update the user's accessibility preferences"
---

<Warning>
  Both endpoints require a valid JWT Bearer token. See [Login](/api/auth/login) to obtain one.
</Warning>

<Note>
  This profile can be auto-populated by the AI accessibility model during the onboarding wizard, then manually overridden in Preferences.
</Note>

## Endpoints

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/api/profile/accessibility` | Retrieve the user's accessibility profile |
| `PUT` | `/api/profile/accessibility` | Update the user's accessibility profile |

---

## GET /api/profile/accessibility

Returns the current accessibility preferences for the authenticated user.

### Response

#### 200 OK

<ResponseField name="theme" type="string" required>
  Active visual theme. One of `standard-accessible`, `large-text-high-contrast`, or `voice-assisted`.
</ResponseField>

<ResponseField name="screenReaderMode" type="boolean" required>
  Whether screen reader mode is enabled.
</ResponseField>

<ResponseField name="fontScale" type="number" required>
  Font scale multiplier currently applied (e.g., `1.0`, `1.2`, `1.6`).
</ResponseField>

<ResponseField name="nudgingLevel" type="string" required>
  Guidance nudging intensity. One of `low`, `medium`, or `high`.
</ResponseField>

<ResponseField name="voiceFeedback" type="boolean" required>
  Whether voice feedback is enabled.
</ResponseField>

### Example

<CodeGroup>

```bash cURL
curl -X GET http://localhost:5000/api/profile/accessibility \
  -H "Authorization: Bearer <token>"
```

```json Response
{
  "theme": "large-text-high-contrast",
  "screenReaderMode": false,
  "fontScale": 1.6,
  "nudgingLevel": "high",
  "voiceFeedback": false
}
```

</CodeGroup>

---

## PUT /api/profile/accessibility

Updates the accessibility preferences for the authenticated user. Only provided fields are updated.

### Request body

<ParamField body="theme" type="string">
  Visual theme to apply. Accepted values:

  - `standard-accessible`
  - `large-text-high-contrast`
  - `voice-assisted`
</ParamField>

<ParamField body="screenReaderMode" type="boolean">
  Enable or disable screen reader mode.
</ParamField>

<ParamField body="fontScale" type="number">
  Font scale multiplier (e.g., `1.2` or `1.6`).
</ParamField>

<ParamField body="nudgingLevel" type="string">
  Guidance nudging intensity. Accepted values: `low`, `medium`, `high`.
</ParamField>

<ParamField body="voiceFeedback" type="boolean">
  Enable or disable voice feedback.
</ParamField>

### Response

#### 200 OK

Returns the updated accessibility profile with the same structure as the GET response.

#### Error responses

| Status | Description |
| --- | --- |
| `400 Bad Request` | Validation error (e.g., unrecognized theme value). |
| `401 Unauthorized` | Invalid or expired token. |

### Example

<CodeGroup>

```bash cURL
curl -X PUT http://localhost:5000/api/profile/accessibility \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "theme": "large-text-high-contrast",
    "screenReaderMode": false,
    "fontScale": 1.6,
    "nudgingLevel": "high",
    "voiceFeedback": false
  }'
```

```json Response
{
  "theme": "large-text-high-contrast",
  "screenReaderMode": false,
  "fontScale": 1.6,
  "nudgingLevel": "high",
  "voiceFeedback": false
}
```

</CodeGroup>
