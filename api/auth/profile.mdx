---
title: "User Profile"
description: "Get and update the authenticated user's profile"
---

<Warning>
  Both endpoints require a valid JWT Bearer token. See [Login](/api/auth/login) to obtain one.
</Warning>

## Endpoints

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/api/profile/me` | Retrieve the authenticated user's profile |
| `PUT` | `/api/profile/me` | Update the authenticated user's profile |

---

## GET /api/profile/me

Returns profile data for the currently authenticated user.

### Response

#### 200 OK

<ResponseField name="id" type="string" required>
  UUID of the authenticated user.
</ResponseField>

<ResponseField name="alias" type="string" required>
  Display name for the user (set during registration as `userName`).
</ResponseField>

<ResponseField name="email" type="string">
  User's registered email address.
</ResponseField>

<ResponseField name="phoneNumber" type="string">
  User's phone number, if set.
</ResponseField>

<ResponseField name="preferredLanguage" type="string" required>
  Preferred UI language, e.g., `"es-MX"`.
</ResponseField>

<ResponseField name="demoMode" type="boolean" required>
  Whether the account is in demo/sandbox mode.
</ResponseField>

### Example

<CodeGroup>

```bash cURL
curl -X GET http://localhost:5000/api/profile/me \
  -H "Authorization: Bearer <token>"
```

```json Response
{
  "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "alias": "María González",
  "email": "maria@example.com",
  "phoneNumber": null,
  "preferredLanguage": "es-MX",
  "demoMode": false
}
```

</CodeGroup>

---

## PUT /api/profile/me

Updates the profile of the currently authenticated user. Only provided fields are updated.

### Request body

<ParamField body="email" type="string">
  Updated email address. Must be a valid email format.
</ParamField>

<ParamField body="phoneNumber" type="string">
  Updated phone number. Must be a valid phone format.
</ParamField>

<ParamField body="preferredLanguage" type="string">
  Updated preferred UI language, e.g., `"es-MX"` or `"en-US"`.
</ParamField>

### Response

#### 200 OK

Returns the updated user profile with the same structure as `GET /api/profile/me`.

#### Error responses

| Status | Description |
| --- | --- |
| `400 Bad Request` | Validation error — invalid email or phone format. |
| `401 Unauthorized` | Invalid or expired token. |

### Example

<CodeGroup>

```bash cURL
curl -X PUT http://localhost:5000/api/profile/me \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "maria.lopez@example.com",
    "phoneNumber": "+52 55 1234 5678",
    "preferredLanguage": "es-MX"
  }'
```

```json Response
{
  "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "alias": "María González",
  "email": "maria.lopez@example.com",
  "phoneNumber": "+52 55 1234 5678",
  "preferredLanguage": "es-MX",
  "demoMode": false
}
```

</CodeGroup>
