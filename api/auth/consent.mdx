---
title: "Consent Management"
description: "Get and update the user's data processing consent"
---

<Warning>
  Both endpoints require a valid JWT Bearer token. See [Login](/api/auth/login) to obtain one.
</Warning>

<Note>
  Consent management aligns with **LFPDPPP** (Ley Federal de Protección de Datos Personales en Posesión de los Particulares — Mexico's Personal Data Protection Law). Users can revoke consent at any time. When `analyticsConsent` is `false`, analytics events are not stored.
</Note>

## Endpoints

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/api/profile/consent` | Retrieve the user's current consent settings |
| `PUT` | `/api/profile/consent` | Update the user's consent settings |

---

## GET /api/profile/consent

Returns the current data processing consent settings for the authenticated user.

### Response

#### 200 OK

<ResponseField name="analyticsConsent" type="boolean" required>
  Whether the user has consented to interaction analytics collection.
</ResponseField>

<ResponseField name="accessibilityDataConsent" type="boolean" required>
  Whether the user has consented to processing of their accessibility profile data.
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp of the last consent update.
</ResponseField>

### Example

<CodeGroup>

```bash cURL
curl -X GET http://localhost:5000/api/profile/consent \
  -H "Authorization: Bearer <token>"
```

```json Response
{
  "analyticsConsent": true,
  "accessibilityDataConsent": true,
  "updatedAt": "2026-03-01T10:00:00Z"
}
```

</CodeGroup>

---

## PUT /api/profile/consent

Updates the data processing consent settings for the authenticated user.

### Request body

<ParamField body="analyticsConsent" type="boolean">
  Consent to collect interaction analytics. When set to `false`, analytics events are no longer stored.
</ParamField>

<ParamField body="accessibilityDataConsent" type="boolean">
  Consent to process accessibility profile data for personalization.
</ParamField>

### Response

#### 200 OK

Returns the updated consent record with the same structure as the GET response.

#### Error responses

| Status | Description |
| --- | --- |
| `400 Bad Request` | Validation error. |
| `401 Unauthorized` | Invalid or expired token. |

### Example

<CodeGroup>

```bash cURL
curl -X PUT http://localhost:5000/api/profile/consent \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "analyticsConsent": false,
    "accessibilityDataConsent": true
  }'
```

```json Response
{
  "analyticsConsent": false,
  "accessibilityDataConsent": true,
  "updatedAt": "2026-03-18T09:30:00Z"
}
```

</CodeGroup>
